Step 1: Dataset Summary & Exploration¶

The pickled data is a dictionary with 4 key/value pairs:

  • 'features' is a 4D array containing raw pixel data of the traffic sign images, (num examples, width, height, channels).
  • 'labels' is a 1D array containing the label/class id of the traffic sign. The file signnames.csv contains id -> name mappings for each id.
  • 'sizes' is a list containing tuples, (width, height) representing the original width and height the image.
  • 'coords' is a list containing tuples, (x1, y1, x2, y2) representing coordinates of a bounding box around the sign in the image. THESE COORDINATES ASSUME THE ORIGINAL IMAGE. THE PICKLED DATA CONTAINS RESIZED VERSIONS (32 by 32) OF THESE IMAGES

Priors¶

Dataset Origin¶

First, I've investigated the origins of the dataset - http://benchmark.ini.rub.de/?section=gtsrb&subsection=dataset. It looks like it was originally much more extensive, I'm guessing it was pre-processed by Udacity to fit students needs better.

Non-local Information¶

There's an international convention for traffic signs and rules: https://en.wikipedia.org/wiki/Vienna_Convention_on_Road_Signs_and_Signals#Road_signs

There's a lot of data there which can't be learned from the dataset: categories, borders, shapes, colors, etc. One particulary important conclusion: signs differ between countries and regions in geometry and color. Consequentially, our classifier will be overfitted to the German convention. Furthermore, there are signs which differ ONLY in one dimension such as chroma, intensity, geometry and sometimes even interpertation (e.g MPH vs KPH speed limit). Therefore, while various dimensionality reduction and downsampling may vastly improve performance and accuracy (even in the test), for some theoretical real world usage it might be unwise to discard these data.

Imputation¶

We've seen the classes have an unbalanced sample quantity. We'll generate some by slightly deforming existing samples. The deformations are: blurring, rotation, scaling, shear, perpective.

In [5]:
# calculate the sample ranges 
rngs = {i:np.argwhere(y_train==i).flatten() for i in range(n_classes)}    
counts = {i: len(rngs[i]) for i in range(n_classes)}
max_count = max([counts[i] for i in range(n_classes)])


tot_count = sum([max_count-counts[i] for i in range(n_classes)])
deformed=np.empty((tot_count,32,32,3), dtype=np.uint8)
deformed_labels = np.empty(tot_count, dtype=np.uint8)

# rotate and scale down matrices
rot_cw = cv2.getRotationMatrix2D((16,16),10,0.9)
rot_ccw = cv2.getRotationMatrix2D((16,16),-10,0.9)
# perpesctive transform matrix
wrp_prsp = np.asarray([[ 0.9,  0.2, -2.0],[ 0.2, 1.0,  -1.0]])

last = 0
for i in range(n_classes):
    rnd = np.random.choice(rngs[i],size=max_count-counts[i])
    for r in rnd:
        if last%5 == 0:
            # avg blur 
            deformed[last] = cv2.blur(X_train[r],(3,3))
        elif last%5 == 1:
            # median blur 
            deformed[last] = cv2.medianBlur(X_train[r],3)
        elif last%5 == 2:
            # rotate clockwise degrees
            deformed[last] = cv2.warpAffine(X_train[r],rot_cw,(32,32))
        elif last%5 == 3:
            # rotate counter-clockwise degrees
            deformed[last] = cv2.warpAffine(X_train[r],rot_ccw,(32,32))        
        elif last%5 == 4:
            deformed[last] = cv2.warpAffine(X_train[r],wrp_prsp,(32,32))
        deformed_labels[last] = i
        last+=1
        
X_train = np.append(X_train,deformed,axis=0)
y_train = np.append(y_train,deformed_labels)
n_train = y_train.shape[0]

Dimensionality Reduction¶

Judging by the images, red and blue hold the most information. Yellow shows up once, but that sign has unique geometry anyway. Green is mostly background vegetation, doesn't show up on the signs themselves. We show, by way of principal component analysis, the red color channel contains the vast majority of information, therefore we discard the green and blue channels.

Step 3: Model Architecture¶

Overview¶

It's basically LeNet with modifications:

  • It looked like some features were smaller than 5x5, so first layer filters are now 3x3.
  • It looked liked the downsampling was happening too fast, so I've added a convolutional layer and more filters.
  • After expanding the convolutional layers, it now appeared like the fully connected layers were too small and few. Hence, I've added an extra layer.
  • The expanded network seemed much more complex, and therefore prone to overfit. I've added dropout layers to both types of layers, with a different probability for each type.
  • Learning rate and batch size were reduced after noticing the training converges much more smoothly and consistently with these values values.
  • Epochs set to 64 as that's about the amount of time it takes the network to converge

The above steps culminated in an ~4% validation accuracy over LeNet with same data and preprocessing.

Layers¶

input: 32x32x1

  1. Convolutional -> ReLU -> dropout -> (32,32,8)
  2. Convolutional -> ReLU -> max pool -> dropout -> (14,14,16)
  3. Convolutional -> ReLU -> max pool -> dropout -> (6,6,32) -> flatten -> (1152)
  4. Fully connected -> ReLU -> dropout -> (512)
  5. Fully connected -> ReLU -> dropout -> (256)
  6. Fully connected -> ReLU -> dropout -> (128)
  7. Fully connected -> output -> (43)

def RoNet(x):  
    # Arguments used for tf.truncated_normal, randomly defines variables for the weights and biases for each layer
    mu = 0
    sigma = 0.1
    
    # Layer 0: Convolutional
    conv0_W = tf.Variable(tf.truncated_normal(shape=(3, 3, 1, 8), mean = mu, stddev = sigma))
    conv0_b = tf.Variable(tf.zeros(8))
    conv0   = tf.nn.conv2d(x, conv0_W, strides=[1, 1, 1, 1], padding='SAME') + conv0_b
    conv0 = tf.nn.relu(conv0)
    conv0 = tf.nn.dropout(conv0, keep_p_conv)
    
    # Layer 1: Convolutional
    conv1_W = tf.Variable(tf.truncated_normal(shape=(3, 3, 8, 16), mean = mu, stddev = sigma))
    conv1_b = tf.Variable(tf.zeros(16))
    conv1   = tf.nn.conv2d(conv0, conv1_W, strides=[1, 1, 1, 1], padding='VALID') + conv1_b
    conv1 = tf.nn.relu(conv1)
    conv1 = tf.nn.max_pool(conv1, ksize=[1, 3, 3, 1], strides=[1, 2, 2, 1], padding='VALID')
    conv1 = tf.nn.dropout(conv1, keep_p_conv)
                           
    # Layer 2: Convolutional
    conv2_W = tf.Variable(tf.truncated_normal(shape=(3, 3, 16, 32), mean = mu, stddev = sigma))
    conv2_b = tf.Variable(tf.zeros(32))
    conv2   = tf.nn.conv2d(conv1, conv2_W, strides=[1, 1, 1, 1], padding='VALID') + conv2_b
    conv2 = tf.nn.relu(conv2)
    conv2 = tf.nn.max_pool(conv2, ksize=[1, 2, 2, 1], strides=[1, 2, 2, 1], padding='VALID')
    conv2 = tf.nn.dropout(conv2, keep_p_conv)
    
    # Layer 3: Fully Connected            
    flat  = flatten(conv2)
    fc1_W  = tf.Variable(tf.truncated_normal(shape=(1152, 512), mean = mu, stddev = sigma))
    fc1_b  = tf.Variable(tf.zeros(512))
    fc1    = tf.matmul(flat, fc1_W) + fc1_b
    fc1    = tf.nn.relu(fc1)
    fc1    = tf.nn.dropout(fc1, keep_p_fc)
    
    # Later 4: Fully Connected
    fc2_W  = tf.Variable(tf.truncated_normal(shape=(512, 256), mean = mu, stddev = sigma))
    fc2_b  = tf.Variable(tf.zeros(256))
    fc2    = tf.matmul(fc1, fc2_W) + fc2_b
    fc2    = tf.nn.relu(fc2)
    fc2    = tf.nn.dropout(fc2, keep_p_fc)
    
    # Layer 5: Fully Connected
    fc3_W  = tf.Variable(tf.truncated_normal(shape=(256, 128), mean = mu, stddev = sigma))
    fc3_b  = tf.Variable(tf.zeros(128))
    fc3 = tf.matmul(fc2, fc3_W) + fc3_b
    fc3    = tf.nn.relu(fc3)
    fc3    = tf.nn.dropout(fc3, keep_p_fc)
    
    # Layer 6: Fully Connected - output
    fc4_W  = tf.Variable(tf.truncated_normal(shape=(128, 43), mean = mu, stddev = sigma))
    fc4_b  = tf.Variable(tf.zeros(43))
    logits = tf.matmul(fc3, fc4_W) + fc4_b
    
    return logits

def evaluate(X_data, y_data):
    num_examples = len(X_data)
    total_accuracy = 0
    sess = tf.get_default_session()
    for offset in range(0, num_examples, BATCH_SIZE):
        batch_x, batch_y = X_data[offset:offset+BATCH_SIZE], y_data[offset:offset+BATCH_SIZE]
        accuracy = sess.run(accuracy_operation, feed_dict={x: batch_x, y: batch_y, keep_p_conv:1.0, keep_p_fc:1.0})
        total_accuracy += (accuracy * len(batch_x))
    return total_accuracy / num_examples
